## Multi-Agent Security Deep Dive (late night session)
- João asked about restricting message tool per-agent → explained 3 approaches (no message tool, binding-only, sandbox)
- Clarified: agents always have read/write/edit on their workspace (runtime base, not a toggleable tool)
- Each agent gets isolated workspace (e.g. `/home/joaohts/.openclaw/agents/<id>/workspace/`)
- **Key insight shared with João**: WITHOUT sandbox, `read` tool accepts absolute paths — can read any file the OS user can access (e.g. `/etc/passwd`, personal files)
- Sandbox mode (`sandbox.mode: "all"`) + Docker is the real isolation layer
- `workspaceAccess: "none"|"ro"|"rw"` controls workspace access inside sandbox
- For a secure public-facing bot: sandbox + tools.allow restrictive + no exec + binding to specific number
- João is learning OpenClaw's security model for potential multi-agent deployment
