Major overnight improvement to the Grafana logging/observability system.

### What Was Done
1. **Fixed all line_format templates** — The #1 issue: all 6 dashboards with log panels showed empty content because `| json` auto-flatten doesn't work with nested arrays. Fixed by using explicit `| json field="message.content[0].text"` syntax with commas between fields.

2. **Fixed gateway log rendering** — Gateway logs use tslog format with numeric keys (`"0"`, `"1"`) and `_meta` nesting. Updated Promtail to extract `level` from `_meta.logLevelName` and module via regex. Dashboard queries use dual-regexp to extract messages from both field `"0"` (error messages) and `"1"` (detail messages).

3. **Improved Promtail config** — Better tool_name extraction handling both `content[0]` and `content[1]` positions (assistant messages have [text, toolCall] structure).

4. **Dashboard improvements:**
   - Dynamic tool_name variable (from labels, not hardcoded)
   - Sub-agent stats use tool_name label instead of fragile text grep
   - Session_id filter added to Errors and Timeline
   - Session list table in Session Explorer
   - Stat panels + Message Flow chart in Messages dashboard
   - Tool Distribution chart + Failed Tools stat in Tools
   - Spawn Timeline in Sub-agents
   - Tool Commands now show arguments
   - Hardcoded `[1h]` replaced with `$__range`
   - Sub-agent launches/results show clean task descriptions

5. **5 critic cycles** — Spawned 5 independent critic sub-agents. Got reports from cycles 1-4 (cycle 5 still running). Scores improved from 3.7/10 to ~7/10 average.
