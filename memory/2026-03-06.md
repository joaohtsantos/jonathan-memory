## Segura Intelligence — Conversation Streaming Refactor
- João planejou uma refatoração do streaming de conversas no Segura Intelligence
- Arquitetura: Temporal → Redis (buffer) → API (SSE) → Front
- `/stream` dispara o Temporal e inicia SSE; `/resubscribe` reenvia tudo desde o início
- Redis Streams (`XADD`/`XREAD BLOCK`/`XRANGE`) como transporte efêmero
- Key pattern: `{agent_id}-{conversation_id}`
- Front salva mensagens no banco num bloco só no fim (front é dono da persistência)
- Cleanup: API deleta key após enviar done (opção A, sem ack — aceita risco mínimo de perda)
- Erro no Temporal: escreve `{type: "error"}` no Redis após esgotar retries
- Resubscribe como rotina ao abrir conversa (check on open)
- Decisões pendentes: formato chunks, TTL, auth, rate limiting, agentes paralelos
